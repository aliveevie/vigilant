# Build a Web3 dApp frontend for "Vigilant" — an agentic onchain insurance protocol on Somnia testnet

## What Vigilant is
A decentralized smart-contract insurance protocol. Underwriters deposit capital into three risk-tranched vaults (A=low risk/low yield, B=medium, C=high risk/high yield) to earn premiums. Policyholders buy parametric exploit coverage. Sentinels earn bounties for flagging live exploits. Risk scores and claim verdicts are decided by Somnia's onchain LLM Inference base agent (deterministic Qwen3-30B inference reaching validator consensus) — not human adjusters or off-chain oracles.

## Tech stack
Vite + React + TypeScript, wagmi + viem for chain interaction, TanStack Query for caching, RainbowKit (with injected MetaMask fallback) for wallet connect.

## Design language
Dark, professional fintech aesthetic — a cross between a modern DeFi dashboard and a Bloomberg terminal. Deep navy/charcoal background, electric-teal and amber accents, monospace font for addresses/hashes/amounts, clean data tables, subtle glow on active/hover states, smooth micro-transitions. Fully responsive (desktop-first, works on mobile). Accessible color contrast.

## Three role-based views behind a shared app shell
App shell: top nav with the Vigilant logo, three tabs (Underwrite / Coverage / Sentinel), a wallet connect button, the connected STT balance, and the network-guard banner.

### 1. Underwriter Dashboard
Deposit/withdraw capital across the three tranches. For each tranche (0=A, 1=B, 2=C) show: total assets, locked capital, share price, premium multiplier (bps), and the connected user's position (vVIG-{A,B,C} share balance + current redeemable STT value).
- Read: `CoverageVault.trancheTotals(uint8)`, `shareToken(uint8)`, `previewDeposit(uint8,uint256)`, `previewRedeem(uint8,uint256)`, `totalCapital()`, `totalLocked()`. Share balance via the vVIG ERC-20 `balanceOf`.
- Write: `deposit(uint8 tier)` payable (msg.value = STT deposited); `withdraw(uint8 tier, uint256 shares)`.

### 2. Policyholder Console
Two-step coverage purchase:
- **Step A — request AI risk score:** call `PolicyManager.requestRiskScore(address coveredContract)` payable. Read the required deposit first from `quoteRiskScoreDeposit()` and use it as msg.value. The request tx returns immediately; the AI agent answers a consensus round later. Poll `coveredContractTier(address)` (returns tier 0/1/2, score 0–100, cache block, expiry block, rationale hash) until it populates, showing a "waiting for agent consensus" pending state.
- **Step B — buy coverage:** once a tier is cached, build and EIP-712-sign a `RiskPolicy` struct, then call `issue(RiskPolicy p, bytes signature)` payable (msg.value = premium). The signed tier must equal the cached tier.
- Show the user's policies: `policiesOf(address)` returns uint256[] ids; `policies(uint256 id)` returns the policy struct (state, holder, coveredContract, coverageAmount, riskTier, startBlock, endBlock, ...).
- **File a claim:** `IncidentResolver.fileClaim(uint256 policyId, bytes32 exploitTx, uint256 incidentBlock)` payable (value from `quoteClaimDeposit()`). Track claim status by reading `claims(uint256)` (state: Pending/Confirmed/Rejected/Escalated). Add an **Escalate** button → `escalate(uint256 claimId)` payable (value from `quoteEscalationDeposit()`).

**EIP-712 details for signing the RiskPolicy:**
- Domain: name `"Vigilant"`, version `"1"`, chainId `50312`, verifyingContract = PolicyManager address. (Verify against `PolicyManager.domainSeparator()`.)
- Primary type `RiskPolicy` with fields in this exact order: `address policyholder`, `address coveredContract`, `uint256 coverageAmount`, `uint8 riskTier`, `uint256 premium`, `uint64 startBlock`, `uint64 endBlock`, `uint256 nonce`.
- The nonce must be strictly greater than `usedNonces(policyholder)`. `startBlock` must be >= current block; `endBlock` > `startBlock`.

### 3. Sentinel Terminal
- Submit a warning: `SentinelRegistry.submitWarning(address coveredContract, bytes32 evidenceTx, uint256 incidentBlock)` payable (value from `quoteWarningDeposit()`).
- Show the connected sentinel's reputation: `getScore(address)` (uint256) and `getHistory(address)` (array of {kind, blockNumber, scoreDelta}).
- List the sentinel's submitted warnings via the `warnings(uint256)` mapping + `WarningSubmitted`/`WarningResolved`/`WarningBountyPaid` events, showing each verdict (Confirmed pays a bounty + pauses coverage; Unconfirmed forfeits the deposit).

## UX requirements
- Every payable action reads its on-chain quote first and prefills msg.value.
- Every tx shows a toast with a clickable link to `https://shannon-explorer.somnia.network/tx/{hash}`.
- AI-decided actions (risk score, claim, warning) are async: the request tx returns instantly, then the agent callback lands ~one consensus round later. Build live-polling pending states for each.
- Proper loading / empty / error states everywhere. Format STT with viem `formatEther`; truncate addresses/hashes as `0x1234…abcd` in mono font.
- Organize chain logic into hooks: `useVault`, `usePolicy`, `useSentinel`, plus a shared `useVigilantConfig` holding addresses + ABIs.
